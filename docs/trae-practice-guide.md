# TRAE 实践过程整理指南

> 本文档帮助你整理 TRAE 开发过程中的 Session ID 和关键截图，用于填写比赛作品帖的「TRAE 实践过程」部分。

---

## 一、如何找到 Session ID

### 什么是 Session ID

Session ID 是 TRAE 中每个对话会话的唯一标识符。比赛要求提供 **≥ 3 个** Session ID，用于验证你确实使用 TRAE 完成了项目开发。

### 查找方法

**方法 1：在 TRAE 界面中查看**

1. 打开 TRAE AI 编程助手
2. 查看左侧「会话历史」或「聊天列表」
3. 找到对应的会话，点击进入
4. 在浏览器地址栏中查看 URL，URL 中类似 `session/xxxxx` 的部分，`xxxxx` 就是 Session ID
5. 或者在会话详情页中查找「会话 ID」「分享」「更多」等按钮，通常能找到复制 Session ID 的选项

**方法 2：从浏览器地址栏复制**

1. 在 TRAE Web 端打开一个会话
2. 地址栏的 URL 通常长这样：`https://trae.ai/session/abc123xyz`
3. 其中 `abc123xyz` 就是 Session ID
4. 直接复制这串字符即可

**方法 3：导出聊天记录时查看**

1. 在 TRAE 中找到「导出」「分享」或「历史记录」功能
2. 导出的文件或分享链接中通常包含 Session ID

## 二、如何截图关键步骤

比赛要求提供 **≥ 3 张** 关键步骤截图，用于展示你用 TRAE 开发的过程。

### 截图工具推荐

- **Windows**：Win + Shift + S（系统自带截图工具），或 Snipaste
- **macOS**：Cmd + Shift + 4（系统自带），或 Xnip
- **浏览器插件**：Awesome Screenshot、FireShot 等（可以截长图）

### 截图要点

1. **清晰可见**：确保截图中的代码、对话内容清晰可辨
2. **包含上下文**：尽量截取完整的 TRAE 界面，包括左侧会话列表、中间对话区、右侧代码区
3. **突出亮点**：优先截取能体现 TRAE 能力的瞬间，比如：
   - 复杂功能一次生成成功
   - TRAE 帮你排查 bug 的过程
   - 架构设计讨论
   - 代码量很大的文件生成
4. **数量充足**：建议截 5-8 张，从中挑选最好的 3-5 张使用

### 截图保存建议

- 统一保存到 `docs/images/trae-screenshots/` 目录下（如果没有就新建）
- 文件名按阶段命名，比如：`phase1-init.png`、`phase2-holdings.png`、`phase3-mobile.png` 等
- 原图保存，不要压缩太厉害影响清晰度

## 四、怎么把内容填入作品帖

### 步骤 1：收集 Session ID

1. 打开 TRAE，回顾你的开发历史
2. 找到 4 个关键阶段对应的会话（阶段 1-4 各一个）
3. 复制每个会话的 Session ID
4. 记录下来，比如：

```
阶段1（项目初始化）：abc123
阶段2（核心功能开发）：def456
阶段3（UI美化+移动端）：ghi789
阶段4（功能增强+比赛）：jkl012
```

### 步骤 2：挑选并整理截图

1. 从你截的图中挑选最好的 3-5 张
2. 复制到 `docs/images/trae-screenshots/` 目录
3. 重命名为有意义的文件名，比如：
   - `01-project-init.png`
   - `02-holdings-dev.png`
   - `03-ui-beautify.png`
   - `04-mobile-adapt.png`

### 步骤 3：编辑 competition-post.md

打开 `docs/competition-post.md`，找到以下位置进行替换：

#### 替换 Session ID

找到 4 处 `[请替换为阶段X的 Session ID]`，分别替换成你收集的 Session ID。

比如把：
```markdown
> 📌 **Session ID**：`[请替换为阶段1的 Session ID]`
```
改成：
```markdown
> 📌 **Session ID**：`abc123def456`
```

#### 插入截图

找到「关键步骤截图」部分，把表格替换成你的真实截图。

**推荐格式 1：图片网格**

```markdown
| 项目初始化 | 核心功能开发 | UI 美化 |
|---|---|---|
| ![项目初始化](images/trae-screenshots/01-project-init.png) | ![核心功能开发](images/trae-screenshots/02-holdings-dev.png) | ![UI 美化](images/trae-screenshots/03-ui-beautify.png) |
```

**推荐格式 2：单张图片 + 说明**

```markdown
#### 截图 1：项目初始化 & 架构搭建

![项目初始化](images/trae-screenshots/01-project-init.png)

用 TRAE 几句话就搭好了 React + TypeScript + Vite + Cloudflare 的完整架构，省了至少半天时间。

#### 截图 2：五源行情容错算法开发

![五源行情开发](images/trae-screenshots/02-holdings-dev.png)

最复杂的五源并发请求 + 中位数择优算法，TRAE 一次就写对了，还自动加了异常处理和超时控制。

#### 截图 3：移动端 APP 化适配

![移动端适配](images/trae-screenshots/04-mobile-adapt.png)

从桌面端表格布局改造成移动端卡片列表 + 底部 Tab Bar，TRAE 帮我调整了所有 CSS 细节，效果媲美原生 APP。
```

#### 替换体验地址

找到「三、Demo 体验地址」部分，把 `https://wealth-agent.pages.dev` 替换成你实际部署的地址。

#### 填写作者信息

在文件顶部找到 `**作者**：[你的名字/团队名]`，替换成你的真实信息。

### 步骤 4：检查清单

填完后对照以下清单检查一遍：

- [ ] Session ID ≥ 3 个（建议 4 个，每个阶段一个）
- [ ] 关键步骤截图 ≥ 3 张
- [ ] 体验地址已替换为真实链接
- [ ] 作者信息已填写
- [ ] 赛道标签正确（学习工作）
- [ ] 所有图片路径正确，能正常显示
- [ ] Markdown 格式正确，没有语法错误
- [ ] 通读一遍，语句通顺，没有错别字

### 步骤 5：导出/复制帖子内容

1. 确认 `docs/competition-post.md` 内容完整正确
2. 可以直接复制 Markdown 内容到比赛投稿页面
3. 或者用 Markdown 编辑器打开预览，确认效果后再提交

## 五、小贴士：让实践过程更有说服力

### 1. 故事化叙述

不要干巴巴地罗列功能，而是讲一个「从 0 到 1 的故事」：
- 一开始遇到了什么问题
- TRAE 怎么帮你解决的
- 中间踩了什么坑
- 最后效果怎么样

有故事、有细节、有情感，评委更容易打高分。

### 2. 对比前后变化

能用对比图就用对比图：
- 第一版 UI vs 最终版 UI
- 自己写的代码 vs TRAE 优化后的代码
- 开发前的构想 vs 开发后的成品

对比产生冲击力，比单纯描述更有说服力。

### 3. 量化成果

尽量用数字说话：
- 原来要做 3 周，用 TRAE 10 天搞定
- 代码量 X 行，TRAE 生成了 80%
- 5 个数据源，TRAE 帮我设计了容错架构
- 三端统一，代码复用率 95%

数字让成果更具体、更可信。

### 4. 突出 TRAE 的价值

重点描述 TRAE 帮你做了什么，而不是你自己做了什么：
- ❌ 不好的：「我实现了五源行情容错」
- ✅ 好的：「我跟 TRAE 描述了需求，它帮我设计了五源并发 + 中位数择优的架构，还自动加了超时和异常处理，一次就跑通了」

比赛比的是「用 TRAE 创作」，所以一定要突出 TRAE 的作用。

## 六、常见问题

**Q：Session ID 找不到怎么办？**
A：试试在 TRAE 的设置、历史记录、分享功能里找找，或者直接联系 TRAE 官方客服咨询。

**Q：截图不够 3 张怎么办？**
A：现在去补截几张！打开 TRAE，回顾一下开发过程，找几个关键对话截一下。如果会话已经清理了，可以重新模拟几个关键对话场景。

**Q：帖子写多长合适？**
A：建议 2000-4000 字，太短说不清楚，太长没人看。重点突出，详略得当。

**Q：可以放视频吗？**
A：看比赛具体要求。如果允许，录一个 1-3 分钟的演示视频效果会更好。
