## 0. 先和大家打个招呼 👋

- **我是谁：**
  一名 90 后程序员，在深圳做后端开发。工资到手后一直在用各种 App 记股票、基金、余额宝、银行卡、信用卡... 但**用了一堆 App 反而更焦虑了**。打开券商 App 看不到基金、打开支付宝看不到股票、想算一下"我现在到底有多少钱"得切四五个 App 加加减减。

  那一瞬间我意识到：不是"理财"难，是**"看清自己的钱"**难。

- **我是怎么用 TRAE 把 Demo 做出来的：**
  Wealth Agent 这个想法在我脑子里大概转了三年。之前自己也用 Vue + Express 写过一个最简版，但写到一半就卡住了——港股的实时数据接口文档太乱，五大行情源之间切换逻辑写得很乱，更别说把"全资产自动换算成人民币"这种基础功能了，做了一版丑到我自己都不愿意用，就丢在那没动。

  这次再捡起来是因为报名了 TRAE 比赛，索性破罐子破摔把所有问题都丢给 TRAE。

  我跟它说的第一句话是："我想做一个**打开网页就能看到自己到底有多少钱**的东西，不要记账 App 那种又长又难填的表单。"

  然后整个过程就成了一个**我和 TRAE 反复拉扯**的过程：

  - "顶栏那个大盘指数能不能显示得真实一点？" → 它把硬编码的"4200"换成了实时的东财 + 新浪双源行情，顺手把上一交易日收盘价和当前价一起拉回来，再也不显示"涨 4.55 但涨跌幅 0.00%"这种鬼东西了。
  - "我加了一只腾讯 00700，你识别成平安银行了" → 它看了一下，哦原来 5 位数字代码是港股，得加 `/^\d{5}$/` 的判断，然后前后端整套都改了。
  - "为什么我加的基金不显示盈亏？" → 它先查了我用的接口是东财的历史净值，再去找了一个叫"天天基金实时估值"的接口（fundgz.1234567.com.cn），然后说"这个源是盘中的估算值，叠加两个源才是真估值"，然后一版把"今日盈亏"和"持仓收益"两个口径都算清楚了。
  - "手机端打开像一个被压扁的网页，难看死了" → 它说"那就 APP 化吧"，然后一通 CSS 改下来，底部给我加了一个带毛玻璃的 Tab Bar，4 个 Tab + 自定义 SVG 图标 + 金色 active 指示条，iPhone 的安全区也帮我处理了，单手操作真就"想点哪就点哪"。
  - 还有几十次"这里没对齐"、"这个颜色不够高级"、"图标太丑换成更有科技感的"，它每一次都理解了我说的"感觉"然后真的改对了。

  最后那天晚上我点开自己写的这个网站，**第一次觉得"这是我的"**。这种感觉我以前写代码从来没有过。

## 1. Demo 简介

- **是什么：** 一个纯前端的**个人财富管理智能体**（Web App），**已部署上线** 👉 **https://wealth-agent.pages.dev/**

- **面向谁：** 和我一样被"多 App 切来切去算账"折磨的工薪族 / 投资小白 / 有多市场持仓的散户（A 股 + 港股 + 基金 + 现金）

- **3 个真正能解决"算账"这件事的核心功能：**

  ### ① 一页看穿你的钱 — 多市场资产自动换算 CNY
  ![dashboard](占位：dashboard.png)
  **打开就是净资产**，再也不用切 4 个 App。底层做了：
  - A 股 / 港股 / 开放式基金 / 现金 / 负债全分类
  - 港股用港币、加股用美元，全部按实时汇率换算成人民币
  - 5 大行情源容错（东财 → 腾讯 → 新浪 → 网易 → 雅虎），一个源挂了立刻切下一个
  - 顶栏直接显示**上证指数 / 恒生指数**实时涨跌，再也不会出现"+4.55 但 0.00%"的鬼数据

  ### ② 收益口径分清楚 — 今日盈亏 / 持仓收益 / 总收益率 三个维度
  ![portfolio](占位：portfolio.png)
  很多人被"我到底赚了多少"搞晕，是因为把**今日盘中波动**和**持仓成本收益**混在一起。我特意把它们拆成三张卡：
  - 今日盈亏（今天盘中相对昨收的浮盈浮亏）
  - 持仓收益（相对买入成本的累计收益）
  - 总收益率（年化口径）
  持仓列表里每一只都**显示成本价 / 现价 / 盈亏金额 / 盈亏比例**，看一眼就知道"这只该不该继续拿着"。

  ### ③ AI 投顾 — 不是"聊天机器人"，是真能"读你的持仓"
  ![ai-advisor](占位：ai-advisor.png)
  这是我最喜欢的一块。市面上 99% 的"AI 理财"是 chatgpt 套壳，**根本不知道你买了什么**。我做的是：
  - **6 大专业分析场景**：持仓诊断 / 行业分散度 / 资产配置建议 / 加仓减仓策略 / 风险预警 / 收益归因
  - 点开任一场景，**自动把你的真实持仓数据带进 prompt**：成本价、市值、行业、收益率、净资产占比... 一次性给到模型
  - 输出不是"你应该分散投资"这种正确的废话，而是"你科技股占比 78%，半导体单一行业 42%，建议把 10% 调到消费 / 医药"
  - 移动端做了**横向滑动的场景卡片**，单手就能切

- **移动端真的可以打开就用** 👇
  ![mobile](占位：mobile.png)
  底部固定 Tab Bar，4 个入口（首页 / 持仓 / 投顾 / 设置），毛玻璃 + 金色指示条 + 48px 最小点击区，是**真的能装进手机桌面**的体验。

## 2. Demo 创作思路

- **灵感来源：** 上面说了，我自己就是"被 4 个 App 折磨的用户"。

- **想解决的问题：**
  1. **跨市场资产汇总**：A 股 / 港股 / 基金 / 现金 / 负债散在 4+ 个 App 里，根本没有一个"我的总资产"
  2. **多源行情不稳定**：单一接口在盘中经常限流，做金融工具最怕"价格滞后导致决策错误"
  3. **AI 投顾空喊口号**：市面上的 AI 理财助手不读你的真实持仓，输出的都是"教科书式废话"
  4. **移动端体验缺失**：大部分个人理财网站在手机上"能用但难用"，没有按 APP 思路设计

- **为什么做这个方向：**
  我不是为了"再做一个记账 App"，而是想验证一件事——**用 TRAE 一个普通程序员能不能独立做出一个能解决自己真实痛点的产品**。事实是：能。而且比我自己写快 5 倍不止。

- **技术架构：**
  ```
  ┌────────────────────────────────────┐
  │  React 18 + TypeScript + Vite       │ ← 前端 SPA
  │  Ant Design 5 / Zustand / Recharts  │
  └──────────────────┬─────────────────┘
                     │  REST
  ┌──────────────────▼─────────────────┐
  │  Cloudflare Workers Functions      │ ← BFF 层
  │  (functions/api/*)                 │
  └──────────────────┬─────────────────┘
                     │
  ┌────────┬─────────┼─────────┬────────┐
  ▼        ▼         ▼         ▼        ▼
  东方财富  腾讯      新浪      网易    雅虎
  + 天天基金实时估值 API
  + Yahoo Finance（港股 5 位代码）
  + Cloudflare D1（用户持仓持久化）
  ```
  整个项目**0 成本部署在 Cloudflare Pages**，全球 CDN 加速，单月成本 < 1 元。

## 最后

很多人问我"为什么不用 XXX 记账 App"，我每次都说：

**"我不是要记账，我是想'看见我的钱'。"**

Wealth Agent 不替代你的券商，它只做一件事——**把所有 App 里的数字聚成一张图，再让一个读过你真实持仓的 AI 跟你聊几句**。

这次做下来最大的感受是：**TRAE 不是"更快的搜索引擎"，它是"会跟你来回拉扯的搭子"**。

我描述"我想要什么感觉"，它给我 5 个版本让我挑；我吐槽"这个不好看"，它真的听懂了"好看"是什么意思。这种协作感，是我以前一个人闷头写三年代码都没体验过的。

如果你也被"多 App 算账"折磨，欢迎来 https://wealth-agent.pages.dev/ 体验一下。也欢迎在评论区告诉我你最希望它加什么功能——我会挑点赞最多的，下个版本里加。
