# 财富管理智能体 - 比赛作品优化 PRD

## Overview
- **Summary**: 基于 TRAE IDE 开发的 AI 驱动个人/家庭财富管理智能体，覆盖 A股 / 港股 / 基金多市场实时行情，5 源容错数据获取，AI 投顾个性化资产分析，桌面 + 移动 + Web 三端统一体验，零成本部署到 Cloudflare Pages。
- **Purpose**: 解决普通投资者缺乏专业财富管理工具、难以系统性跟踪资产和投资组合的痛点，让每个人都能拥有零成本、跨终端、懂自己的 AI 资产管家。
- **Target Users**: 25-55 岁有投资理财需求的个人和家庭用户，从初入股市的新手到有一定经验的进阶投资者。

## Goals
- **Goal 1 创新价值最大化（30% 权重）**：突出 5 源行情容错、零成本云部署、多市场统一管理、AI 实时上下文投顾四大技术创新点
- **Goal 2 产品完成度拉满（30% 权重）**：四大核心模块全部可用，体验流畅，数据准确，桌面端 + 移动端双端完善
- **Goal 3 TRAE 实践充分展示（25% 权重）**：整理开发过程中的关键 Session、踩坑记录、效率提升点，证明作品由 TRAE 全程完成
- **Goal 4 展示表现力升级（15% 权重）**：高质量 README、精美截图、流畅的体验链接、结构清晰的作品帖

## Non-Goals (Out of Scope)
- 不做后端完整用户系统（保持当前 localStorage + D1 轻量方案，聚焦前端体验）
- 不做交易撮合（只做展示和分析，不涉及真实交易）
- 不做 iOS/Android 原生 App（Web + 移动端响应式 + Electron 足矣）
- 不做复杂的量化策略回测（保持 AI 投顾 + 资产分析定位）

## Functional Requirements

### FR-1: AI 投顾专业分析升级
- 新增 6 大专业分析场景：资产配置体检、风险评估、组合优化建议、持仓集中度分析、收益归因分析、行业分布洞察
- AI 自动读取用户资产 / 持仓数据作为上下文，生成结构化分析报告
- 桌面端左侧场景入口 + 移动端横向滑动场景模板

### FR-2: 数据可视化增强
- 收益走势图（基于持仓成本 vs 当前市值，模拟历史收益曲线）
- 行业分布饼图（按持仓标的所属行业分类）
- 资产类别柱状图对比（当前 vs 目标配置）
- 移动端图表单列全宽 + 可横向滑动

### FR-3: 资产收益追踪
- 总资产收益概览卡（总收益金额 + 收益率）
- 单资产收益明细（成本、当前、盈亏、收益率）
- 收益排行榜（涨幅榜 / 跌幅榜）

### FR-4: 部署稳定性保障
- Cloudflare Pages 部署验证，确保线上体验链接 24/7 可用
- 离线 fallback（localStorage 数据持久化，断网也能用）
- 行情接口超时保护和降级策略

### FR-5: 比赛作品材料准备
- 高质量 README（已完成初版，需补充比赛作品定位）
- 桌面端 + 移动端全套展示截图
- 比赛作品帖正文（按官方模板：简介 + 创作思路 + 体验地址 + TRAE 实践过程）
- TRAE 开发过程 Session ID 整理

## Non-Functional Requirements

### NFR-1: 性能
- 首屏加载 < 2s（Cloudflare CDN）
- 行情刷新 30s 自动同步
- 页面切换动画 < 300ms

### NFR-2: 可靠性
- 5 源行情容错，任一源故障不影响使用
- 30% 价格异常检测，防数据源脏数据
- localStorage 兜底，D1 写入失败不阻塞使用

### NFR-3: 可访问性
- 桌面端 1440p / 1080p 自适应
- 移动端 ≤768px APP 化布局
- 触摸目标 ≥ 44px
- 支持键盘导航

### NFR-4: 可部署性
- 一键部署到 Cloudflare Pages（零配置）
- 无需服务器、无需域名、完全免费

## Acceptance Criteria

### AC-1: AI 投顾 6 大专业分析场景可用
- **Given**: 用户已添加资产和持仓数据
- **When**: 用户点击任一专业分析场景（如「组合优化建议」）
- **Then**: AI 读取实时资产 / 持仓数据，生成结构化分析报告，包含数据引用和具体建议
- **Verification**: `programmatic`
- **Notes**: 至少 6 个场景模板，每个模板有独立的 System Prompt

### AC-2: 收益数据完整展示
- **Given**: 用户有持仓数据（含成本价和数量）
- **When**: 进入资产总览或持仓管理页
- **Then**: 显示总收益金额、总收益率、单标的盈亏、涨跌幅排行榜
- **Verification**: `programmatic`
- **Notes**: 收益 = (当前价 - 成本价) × 数量，收益率 = 收益 / 总成本 × 100%

### AC-3: 行业分布可视化
- **Given**: 用户有多个不同行业的持仓
- **When**: 查看资产分布区域
- **Then**: 展示行业分布环形图，鼠标悬停显示行业名称和占比
- **Verification**: `human-judgment`
- **Notes**: 行业分类可基于标的代码或名称简单映射

### AC-4: 移动端体验完整
- **Given**: 在手机屏幕（≤768px）访问
- **When**: 浏览全部 4 个 Tab
- **Then**: 底部 Tab Bar 固定、卡片化布局、触摸目标 ≥44px、图表单列全宽
- **Verification**: `human-judgment`
- **Notes**: APP 风格布局，非缩小版桌面端

### AC-5: Cloudflare Pages 部署可用
- **Given**: 项目已推送到 GitHub
- **When**: 访问 wealth-agent.pages.dev
- **Then**: 页面正常加载，四大功能模块均可操作，数据持久化正常
- **Verification**: `programmatic`
- **Notes**: 体验链接必须公开、稳定、可重复访问

### AC-6: 比赛作品帖材料齐全
- **Given**: 项目优化完成
- **When**: 准备发布初赛作品帖
- **Then**: 具备：①作品简介 ②创作思路 ③体验地址 ④TRAE 实践过程（≥3 截图 + ≥3 Session ID）
- **Verification**: `human-judgment`
- **Notes**: 按官方模板格式，赛道标签正确（学习工作）

### AC-7: 5 源行情容错机制有效
- **Given**: 持仓包含 A股 / 港股 / 基金
- **When**: 触发行情刷新
- **Then**: 即使 1-2 个数据源失败，仍能获取到正确价格，涨跌幅计算准确
- **Verification**: `programmatic`
- **Notes**: 价格与昨收偏差超过 30% 自动丢弃
