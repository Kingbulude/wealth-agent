# 开发工作流与部署规则

## 代码提交与同步流程

每次完成代码修改后，必须严格按照以下流程执行，不得遗漏任何步骤：

### 1. 本地验证
- 先运行构建命令确保代码能正常编译：`npm run build`
- 对于 TypeScript 项目，必须先运行类型检查：`npx tsc --noEmit`

### 2. 合并到 main 分支
- 所有修改最终必须合并到 `main` 分支
- 禁止只在临时分支（如 `trae/agent-*`）上提交而不合并到 main
- 如果当前不在 main 分支，改完代码后必须：
  ```bash
  git checkout main
  git merge <当前分支>
  ```

### 3. 推送到远程
- 合并到 main 后，必须推送到远程仓库：`git push origin main`
- 必须明确告知用户"已推送到 main 分支，等待部署"

### 4. 部署验证提醒
- 推送后提醒用户等待 Cloudflare Pages（或其他部署平台）自动部署
- 告知大致等待时间（通常 2-3 分钟）
- 告知用户如何验证部署是否成功

## 工作闭环原则

修改代码 → 本地验证 → 合并 main → 推送远程 → 告知用户

以上五步构成完整闭环，缺少任何一步都等于任务未完成。
